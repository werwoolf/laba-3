​Retrieve a Paginated List of Tasks

GET
/v1/tasks

Fetches a list of tasks with pagination and extensive filtering options, including by name, priority, assignee, and date range. Supports sorting by due date or priority. Requires authentication with task read permissions.

​Retrieve a Paginated List of Tasks › query Parameters

  • pageinteger · min: 1 · required

    The page number for pagination (1-based indexing).

  • page_sizeinteger · min: 1 · max: 100 · required

    Number of tasks to return per page.

  • filter_by_namestring

    Filter tasks by partial match on task name.

  • filter_by_prioritystring · enum

    Filter tasks by priority level (normal, high, critical).

    Enum values:
    normal
    high
    critical
  • filter_by_resultstring · enum

    Filter tasks by their completion status (completed or due).

    Enum values:
    completed
    due
  • filter_by_assignee_usersstring

    Comma-separated list of user IDs to filter tasks by assignees.

  • filter_by_registersstring

    Filter tasks by associated register IDs.

  • historyboolean

    If true, includes completed tasks in the results.

  • order_by_due_dateboolean

    If true, sorts tasks by due date in ascending order.

  • order_by_priorityboolean

    If true, sorts tasks by priority (critical > high > normal).

  • from_datestring · date-time

    Filter tasks with due dates on or after this date.

  • to_datestring · date-time

    Filter tasks with due dates on or before this date.

  • all_tasksboolean

    If true, includes all tasks regardless of other filters.

​Retrieve a Paginated List of Tasks › Responses

    • totalinteger · required

      Total number of tasks matching the query.

    • pageinteger · min: 1 · required

      Current page number.

    • page_sizeinteger · min: 1 · required

      Number of tasks per page.

    • dataobject[] · required

      List of tasks with related data.

​Create a New Task

POST
/v1/tasks

Creates a new task with specified details such as name, priority, due date, and assigned users. Supports recurring tasks and reminders. Requires authentication with task creation permissions.

​Create a New Task › Request Body

  • namestring · minLength: 1 · maxLength: 50 · required

    Name or title of the task.

  • prioritystring · enum · required

    Priority level of a task: 'normal' for standard tasks, 'high' for important tasks, 'critical' for urgent tasks requiring immediate attention.

    Enum values:
    normal
    high
    critical
  • first_due_datestring · date-time · required

    Initial due date for the task.

  • is_every_employee_needs_to_completeboolean · required

    If true, the task must be completed by all employees.

  • form_idinteger · min: 1 · required

    Identifier of the form associated with the task.

  • start_datestring · date-time · required

    Date and time when the task becomes active.

  • recurrency_rulestring

    Rule defining the task's recurrence pattern (e.g., 'daily', 'weekly').

  • user_idsinteger[]

    List of user IDs assigned to the task.

    Default: []
  • users_overdue_subsinteger[]

    List of user IDs subscribed to overdue notifications.

    Default: []
  • users_completed_subsinteger[]

    List of user IDs subscribed to completion notifications.

    Default: []
  • descriptionstring

    Detailed description of the task's purpose or requirements.

  • due_remaindersobject[]

    List of reminders to send before or after the due date.

    Default: []

​Create a New Task › Responses

    • taskobject · required

      Represents a task's core details, used in responses for task creation and updates.

​Create a Task Result

POST
/v1/tasks/result

Records the result of a task, such as completion status and form data. Requires authentication with task update permissions.

​Create a Task Result › Request Body

  • task_idinteger · min: 1 · required

    Identifier of the task.

  • resultstring · enum · required

    Status of a task: 'completed' for finished tasks, 'due' for tasks pending completion.

    Enum values:
    completed
    due
  • form_resultsobject · required

    Data submitted for the task's associated form.

  • task_result_idinteger · min: 1

    Optional identifier for an existing task result, if updating.

  • registersobject[]

    List of register results associated with the task.

    Default: []

​Create a Task Result › Responses

    • task_resultobject · required

      Represents the result of a task, including completion details and submitted form data.

POST /v1/tasks/result
