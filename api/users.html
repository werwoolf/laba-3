​users

​Retrieve a Paginated List of Users

GET
/v1/users

Fetches a list of users with pagination and optional filtering. Supports excluding specific users and filtering by name. Requires authentication with appropriate permissions.

​Retrieve a Paginated List of Users › query Parameters

  • pageinteger · min: 1 · required

    The page number for pagination (1-based indexing).

  • page_sizeinteger · min: 1 · max: 100 · required

    Number of users to return per page.

  • excludestring

    Comma-separated list of user IDs to exclude from the results.

  • filter_by_namestring

    Filter users by partial match on first or last name.

​Retrieve a Paginated List of Users › Responses

    • totalinteger · required

      Total number of users matching the query.

    • pageinteger · min: 1 · required

      Current page number.

    • page_sizeinteger · min: 1 · required

      Number of users per page.

    • dataobject[] · required

      List of users.

​Sign Up a New User

POST
/v1/users

Creates a new user account with the specified details. Requires authentication and admin permissions unless used in a public registration flow.

​Sign Up a New User › Request Body

  • first_namestring · minLength: 1 · maxLength: 100 · required

    The user's first name.

  • last_namestring · minLength: 1 · maxLength: 100 · required

    The user's last name.

  • emailstring · email · required

    The user's email address.

  • passwordstring · password · minLength: 8 · maxLength: 300 · writeOnly · required

    The user's password.

  • rolestring · enum · required

    Role assigned to a user: 'admin' for users with elevated permissions, 'user' for standard users.

    Enum values:
    admin
    user
  • redirect_urlstring

    Optional URL to redirect the user after signup, if applicable.

​Sign Up a New User › Responses

    • userobject · required

      Detailed representation of a user, including authentication and role information.

    • sessionstring · required

      Session token or identifier for the user's session.

​Delete a User

DELETE
/v1/users

Removes a user from the system by their ID. Requires authentication and admin permissions.

​Delete a User › Request Body

  • idinteger · min: 1 · required

    Unique identifier of the user to delete.

​Delete a User › Responses

    • successboolean · required

      True if the user was deleted successfully.
